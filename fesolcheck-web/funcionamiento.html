Patrón

MVC adaptado sin framework. panel.php actúa como front-controller: valida la sesión, determina el módulo activo y carga el fragmento PHP correspondiente.

Sesiones

Sesiones PHP nativas. Control de rol admin / empleado verificado en cada endpoint antes de ejecutar ninguna query. Sin JWT ni tokens.

API REST

15 endpoints internos. Todos devuelven JSON con estructura uniforme:
{success: bool, message: string, data: []}
El frontend consume la API exclusivamente con fetch() nativo.

02
Flujo de datos

Ciclo completo de una acción

Ejemplo: empleado clica "Fichar Entrada" → INSERT en MySQL → DOM actualizado

secciones/api/fichaje.php (fragmento) 
// secciones/api/fichaje.php
session_start();
header('Content-Type: application/json');
require_once '../../config.php'; // $pdo disponible

if (!isset($_SESSION['usuario'])) {
    echo json_encode([
        'success' => false,
        'message' => 'No autorizado'
    ]);
    exit;
}

$tipo      = $_POST['tipo'] ?? ''; // 'entrada'|'pausa'|'salida'
$idUsuario = $_SESSION['usuario']['id'];

$stmt = $pdo->prepare(
    "INSERT INTO FICHAJE (id_usuario, tipo, fecha, hora)
     VALUES (:u, :t, CURDATE(), CURTIME())"
);
$stmt->execute(['u' => $idUsuario, 't' => $tipo]);

echo json_encode([
    'success' => true,
    'hora'    => date('H:i')
]);
Llamada fetch desde el navegador 
// Llamada fetch desde el navegador
async function registrarFichaje(tipo) {
  const res = await fetch(
    'secciones/api/fichaje.php', {
      method: 'POST',
      body: new URLSearchParams({ tipo })
    }
  );
  const data = await res.json();

  if (data.success) {
    document
      .querySelector('.fichaje-hora')
      .textContent = data.hora;

    document
      .querySelector('.fichaje-estado')
      .classList.add('activo');
  }
}
